1.5.0.0
-------
* Speech file viewer. The standalone speech/effects container of the talkie editions (MONSTER.SOU, or the FM Towns-style "game".SOU) now appears as a node in the tree and lists every voice/effect line with its lip-sync timestamp count, sample rate and duration. Each line can be played in the editor and exported to WAV (one entry or all at once). The file is parsed lazily and the audio is read on demand, so the hundreds-of-MB speech files open instantly.

* CD audio viewer. Releases that ship the CD music ripped as CDDA.SOU now show a node listing each audio track with its start frame and duration; tracks can be played and exported to WAV (44100 Hz, 16-bit stereo), one or all at once.

* MIDI sound blocks now play. The iMUSE tracks use the SMF format 2 that the Windows sequencer rejects; a single-track format 2 is patched to format 0 for playback (the exported bytes are untouched), and MCI errors are now reported instead of failing silently.

1.4.0.0
-------
* Full SCUMM v5 support: The Secret of Monkey Island (CD VGA), Monkey Island 2: LeChuck's Revenge and Indiana Jones and the Fate of Atlantis now have the same feature set as the v6 games. Every block type is decoded with its viewer (scripts, objects, fonts, sounds, walk boxes, palettes - including the v5-only EPAL EGA dithering palette), and images, costumes and charsets round-trip byte-identically.

* New SCUMM v5 script disassembler: the complete v5 opcode set (a different language from v6 - parameters carried inline with variable/literal bits, conditionals with embedded jumps, expression blocks) rendered in the same readable C#-like listing. Every script and every object verb script of the three games decodes 100%.

* Game text translation works on v5: export/import with automatic jump and verb-table remapping, validated against all editions of MI1, MI2 and Indy4 (savegame-name strings are excluded from the export). Font (CHAR) PNG editing also works on v5, with the atlas margins now adapting to fonts that use negative glyph offsets (the European MI2 fonts).

* Language detection now covers the v5 games (the French, German, Italian and Spanish editions are recognized).

* CD edition detection: releases that ship ripped CD audio (a CDDA.SOU of 50 MB or more, like the Monkey Island 1 CD) are reported as the "CD" edition - without being confused with a talkie, since CD audio is music, not speech. The status bar now distinguishes Talkie, CD and Floppy.

* More tolerant game loading: unknown blocks inside rooms and disk sections are kept as raw, byte-preserved blocks instead of aborting the load. This makes the fan-made Ultimate Talkie editions of MI1 and MI2 fully loadable and editable.

* The whole editor speaks English now: all remaining Portuguese messages, dialogs, reports and error texts were translated.

1.3.1.0
-------
* Games are now opened by folder: "File > Open Game Folder" detects the game, the SCUMM version and the edition from the content of the selected folder alone (nothing is guessed from folder names or subfolders). The status bar shows the game name, the edition and the SCUMM version.

* Talkie x Floppy edition detection: a release is considered talkie when its speech file (MONSTER.SOU, or the FM Towns-style <game>.SOU) is bigger than 50 MB. The floppy editions also ship a MONSTER.SOU, but it only holds sound effects (a few MB), while real recorded speech takes 150 MB or more - the rule now applies to every game, not only Monkey Island 1.

* Best-effort language detection: the editor samples the game's own texts and recognizes English, Portuguese, Spanish, German, French and Italian, showing the language in the status bar when the detection is confident. Validated against 13 real editions, including the ScummBR translation (Portuguese) and the official French/German/Italian/Spanish releases.

* Added the FM Towns release of Indiana Jones and the Fate of Atlantis (INDY4.000/.001) to the known games.

1.3.0.0
-------
* Game text translation support. "File > Export game texts" writes every translatable string of the game (dialogues, prints, verb names, object names) to a single .txt file: one line per text with a stable id, control codes as readable tokens ({br}, {wait}, {sound:N}...) and accented characters mapped through a charmap stored in the file header. "File > Import game texts" reads the file back and rebuilds the bytecode - jump offsets and the verb offset tables are remapped automatically when a translation is shorter or longer than the original, and every rebuilt script is re-disassembled and verified before being accepted.

* The charmap is under the team's control: the import rebuilds the character mapping from the "; charmap:" header line, so free font slots can be mapped to new characters or used directly in the text as {0xNN} (e.g. custom ligatures like "Cla{0xC1}e"). On export the translator can optionally paste the charmap of an in-progress translation to keep its custom characters. Lines with nothing to translate (empty names, sound-only strings) are left out of the file, and imported texts are validated against the game fonts, reporting characters that have no glyph.

* Charset (font) editing via PNG. Each CHAR block - or all of them at once through "File > Export/Import game fonts" - can be exported as an editable 8-bit indexed PNG atlas (256 slots separated by divider lines) plus a guide image showing the hex id, the origin mark and the current glyph of every slot. Drawing in an empty slot adds the glyph to the font, extending the character table when needed; untouched slots keep their original bytes, and edited glyphs preserve the font metrics (cursor advance / spacing).

* New OBCD viewer: the object properties (code header, name) and the verb table are shown as fields, with the disassembled verb bytecode below and a label at each verb entry point.

* Script listings now render messages with the same friendly tokens and accented characters used by the text export, instead of raw escape bytes.

* Index file viewers for MAXS (engine limits), DOBJ (object owners/states/classes), AARY (array directory - previously missing from the tree) and RNAM (room names, populated on SCUMM v5 games); the palette offset block (OFFS) also gained a viewer.

1.1.1.0
-------
* Decoded the room navigation and object-code blocks, each with its own viewer: BOXD (walk boxes), BOXM (box matrix), SCAL (scale slots) and OBCD (object code, showing the object name and verb count).

* Decoded the sound blocks (SOUN): the iMUSE device versions (AdLib, Roland, General MIDI) are listed and can be exported, and Standard MIDI / digital (VOC) sounds can be played.

* Added a SCUMM 6 script disassembler. Global, local, entry and exit scripts (SCRP, LSCR, EXCD, ENCD) are shown as a readable, C#-like listing.

* Decoded the charset/font blocks (CHAR): the glyphs are rendered in a font atlas viewer.

* With this every SCUMM 6 block type is now parsed - there are no more raw, "not implemented" blocks. Validated against Day of the Tentacle and Sam & Max with a byte-identical round-trip.

1.1.0.0
-------
* Lossless indexed image export/import. Backgrounds and objects are now exported as 8-bit indexed PNGs that store the original palette index of every pixel, and importing reads those indexes back directly instead of guessing them from the pixel color. This preserves the exact index even when the palette has repeated colors, so the image keeps rendering correctly under any of the alternate (APAL) palettes. The old color-matching import and the .idx side files are no longer needed.

* Costumes are exported with their own (costume-local) palette and their relative indexes, also round-tripping losslessly.

* Added detection of Sam & Max installs that use SAMNMAX.SM0 / SAMNMAX.SM1.

* Migrated the editor to .NET 10 and Visual Studio 2022 (SDK-style projects). It now builds and runs on the modern .NET runtime.
